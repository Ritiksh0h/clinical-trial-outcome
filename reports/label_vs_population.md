# Label Effect vs Population Effect

Generated by `python scripts/label_vs_population.py`. Diagnostic only.

**Question:** the weak set is ~83% positive but the gold set is ~20% positive (Phase I). Is that gap because the weak *labels* are wrong, or because the weak set is a different *population* of trials than the gold set?

**Method:** gold trials are routed to a phase by CTO canonical membership (the nct_id appears in `cto_phase{N}.parquet`). On the overlap (trials in both the weak phase file and the gold set) we have *two labelings of the same trials*, so population is held constant and the residual gap is the pure label effect.

| Phase I | 41,504 | 3,239 | 29.4% |

- Gold trials in **2** weak phase files (phase-combo trials, e.g. PHASE1/PHASE2): **1,412** — these appear in both phases' overlaps, matching how CTO assigns them.

## Part C — Decomposition: population effect vs label effect

|-------|-----------|------------------|--------------|-------------|--------------|-------------------|--------------|

### Conclusion

Averaged across phases, the population effect is **45.5%** and the label effect is **3.0%** — the population effect is **94%** of the total gap.

**The gap is population-driven, not label-driven.** On the same trials, the CTO weak labels agree with the expert gold labels ~96%+ of the time and the positive rates differ by only 2–4pp. The weak labeling function is *not* systematically optimistic. The 83%→20% gap exists because the full weak training set is dominated by trials that COMPLETED (which both labelings call success), while the gold set is enriched for TERMINATED / WITHDRAWN trials (which both labelings call failure).

**Implication for Phase 2.** The Phase 1 model did not "learn labeling-function artifacts." It learned on a population that is ~83% completed and was evaluated on a population that is ~20% completed — a covariate/prior shift, not label noise. Training on gold labels alone fixes the *evaluation population* mismatch, but the underlying target both label sets encode is still **completion vs termination**, not efficacy (see `reports/definition_audit.md`).
